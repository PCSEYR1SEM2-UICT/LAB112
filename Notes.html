COURSE 01
C# Programming
Variables
What is a Variable? Covered

A variable is a container that stores a value in your program. Think of it like a labelled box — the label is the variable name, and whatever is inside the box is the value.

Real life example

Imagine a student register. Each student has a name, an age, and a score. In a program, each of those would be a separate variable holding a different value.

In C# you must tell the program what type of data the variable will hold before using it. This is called declaring a variable.

// Syntax: dataType variableName = value;
int age = 20;
string name = "Alice";
double score = 85.5;
bool isPassed = true;

Common data types in C#:

int — whole numbers like 1, 20, 100
double — decimal numbers like 3.14, 85.5
string — text like "Hello" or "Alice"
bool — only true or false
char — a single character like 'A' or '9'

Good to know

Variable names cannot start with a number, cannot have spaces, and are case sensitive — meaning age and Age are treated as two completely different variables.

Displaying Text
Console.Write and Console.WriteLine Covered

To display text or values on the screen in C#, we use Console.WriteLine() or Console.Write().

The difference: WriteLine prints the text and then moves to a new line. Write prints the text and stays on the same line.

// WriteLine — moves to next line after printing
Console.WriteLine("Hello, World!");
Console.WriteLine("Welcome to LAB 112");

// Output:
Hello, World!
Welcome to LAB 112

// Write — stays on same line
Console.Write("Hello, ");
Console.Write("World!");

// Output:
Hello, World!

You can also display the value of a variable by placing it inside the brackets:

string name = "Alice";
int age = 20;

Console.WriteLine(name);
Console.WriteLine("Age: " + age);

// Output:
Alice
Age: 20
String Interpolation — cleaner way

Instead of joining with +, you can use $ before the string and put variables in {}. This is called string interpolation and is much cleaner:

string name = "Alice";
int age = 20;

Console.WriteLine($"My name is {name} and I am {age} years old.");

// Output:
My name is Alice and I am 20 years old.
Comments
What are Comments? Covered

A comment is a note you write inside your code that the computer completely ignores. It is only there for you or other developers reading the code to understand what is happening.

Real life example

Think of comments like the notes a student writes in the margin of a textbook. The notes don't change the textbook content — they're just there to help the reader understand.

// This is a single line comment

int age = 20; // This stores the student's age

/* This is a
   multi-line comment.
   Use this for longer explanations. */

Console.WriteLine(age); // This prints the age
Why comments matter

Good code with good comments is something any developer can pick up and understand. When you come back to your own code weeks later, comments remind you why you wrote something a certain way.

Escape Sequences
Special Characters in Strings Covered

Sometimes you need to include special characters inside a string — like a new line, a tab space, or even a quotation mark. You can't just type them directly so you use escape sequences which start with a backslash \.

// \n — new line
Console.WriteLine("Hello\nWorld");
Hello
World

// \t — tab space
Console.WriteLine("Name:\tAlice");
Name:    Alice

// \" — quotation mark inside a string
Console.WriteLine("She said \"Hello!\"");
She said "Hello!"

// \\ — backslash itself
Console.WriteLine("C:\\Users\\Alice");
C:\Users\Alice
Arrays
Storing Multiple Values Covered

An array is a variable that can hold multiple values of the same type in one place, instead of creating a separate variable for each value.

Real life example

Instead of having seat1, seat2, seat3... in a classroom, you just have one seats[] array that holds all of them. A shelf of books is an array — one shelf, many books, each at a specific position.

// Declaring an array — dataType[] arrayName = new dataType[size];
int[] scores = new int[3];
scores[0] = 85;
scores[1] = 90;
scores[2] = 78;

// Shorter way — declare and fill at once
int[] scores = { 85, 90, 78 };

// Accessing values — index starts at 0
Console.WriteLine(scores[0]); // prints 85
Console.WriteLine(scores[2]); // prints 78
Important — Index starts at 0!

The first item is always at position 0, not 1. So in an array of 3 items, the positions are 0, 1, and 2. Trying to access position 3 would cause an error.

You can loop through an array using a for loop:

string[] names = { "Alice", "Bob", "Charlie" };

for (int i = 0; i < names.Length; i++)
{
    Console.WriteLine(names[i]);
}

// Output:
Alice
Bob
Charlie
Loops
Repeating Actions Covered

A loop allows you to repeat a block of code multiple times. This saves you from writing the same code over and over again.

Real life example

Imagine a music playlist set to Repeat. It plays the songs (the code) until the playlist ends, and then starts again. It keeps looping until you hit stop (the condition becomes false).

There are four main types of loops in C#:

1. The 'for' Loop

Use this when you know exactly how many times you want to loop. The for loop has three distinct parts that control its flow:

  • Start (int i = 1): This creates a counter variable. It only runs once at the very beginning.
  • Stop (i <= 3): This is the condition. The loop keeps running as long as this is true. It checks this before every loop.
  • Step (i++): This updates the counter (adds 1). It runs at the end of every loop.
// 1. Start at 1 | 2. Check if <= 3 | 4. Increase by 1
for (int i = 1; i <= 3; i++) 
{
    // 3. Run this code
    Console.WriteLine("Count: " + i);
}
2. The 'while' Loop

Use this when you want to loop as long as a condition is true.

int health = 3;
while (health > 0) 
{
    Console.WriteLine("Still alive!");
    health--; // decreases health
}
3. The 'do-while' Loop

Similar to while, but it always runs at least once before checking the condition.

int attempts = 0;
do 
{
    Console.WriteLine("Trying connection...");
    attempts++;
} while (attempts < 1);
4. The 'foreach' Loop

Use this to easily loop through items in an array or a collection.

string[] cars = {"Volvo", "BMW", "Ford"};
foreach (string i in cars) 
{
    Console.WriteLine(i);
}
Advanced Loop Techniques Covered
Nested Loops (Loops inside Loops)

You can place a loop inside another loop. The "inner loop" will finish all of its iterations for every single iteration of the "outer loop".

for (int i = 1; i <= 2; i++) // Outer loop (runs 2 times)
{
    Console.WriteLine("Outer: " + i);
    for (int j = 1; j <= 3; j++) // Inner loop (runs 3 times per outer)
    {
        Console.WriteLine("  Inner: " + j);
    }
}
// Total inner runs = 2 x 3 = 6 times

How it works:
1. The outer loop starts with i = 1.
2. It prints "Outer: 1".
3. Then, the inner loop starts. It runs completely from j = 1 to j = 3, printing "Inner: 1", "Inner: 2", and "Inner: 3".
4. The inner loop finishes, and the outer loop moves to its second step (i = 2).
5. The inner loop starts completely over again from 1 to 3. Think of it like hours (outer) and minutes (inner) on a clock.

Reversing a Loop

You don't always have to count up using i++. You can count backwards using i--. This is useful for displaying lists from the bottom up.

// Counting backwards from 5 down to 1
for (int i = 5; i > 0; i--) 
{
    Console.WriteLine(i);
}
// Output: 5, 4, 3, 2, 1

How it works:
1. Start: We start with i = 5 instead of 0.
2. Stop: The condition is i > 0. As long as the number is greater than zero, the loop keeps running.
3. Step: Instead of i++ (add 1), we use i-- (subtract 1). It prints 5, subtracts 1 to become 4, prints 4, and so on until it hits 0 and stops.

Interactive Example: Storing Input in an Array

This is a classic program: Ask the user for 3 names, use a loop to store them in an array, and use another loop to print them back out.

string[] students = new string[3];

// 1. Loop to get input and store it
for (int i = 0; i < students.Length; i++) 
{
    Console.Write($"Enter student {i+1} name: ");
    students[i] = Console.ReadLine();
}

Console.WriteLine("\n--- Student List ---");

// 2. Loop to display the stored data
foreach (string name in students) 
{
    Console.WriteLine("Student: " + name);
}

How it works:
Loop 1 (Input): We use a for loop to ask the user for a name 3 times. We use students.Length to ensure we fill the array perfectly. students[i] stores each name into slot 0, then slot 1, then slot 2.
Loop 2 (Output): We use a foreach loop. It automatically grabs each string out of the students array one by one, temporarily calls it name, and prints it out.

Reading Complex Loops

Q: How do I trace a loop without getting confused?
A: Always read a loop like a story. Look at the starting point (int i=0), look at the stopping condition (i < 5), and look at the step (i++). Then, ask yourself: "What happens the very first time? What happens the second time? When does it finally break?" Tracking variables on a piece of paper (a "trace table") makes complex loops easy to understand.

Methods
Reusable Blocks of Code Covered

A method is a block of code that does a specific task and can be called (used) whenever you need it. Instead of writing the same code over and over, you write it once in a method and call it.

Real life example

A printer is like a method. You press Print (call the method), it does its job (runs the code), and gives you the result. You don't need to know exactly how it works inside — you just call it and it delivers.

// Defining a method
static void Greet()
{
    Console.WriteLine("Hello, welcome to LAB 112!");
}

// Calling the method
Greet(); // Output: Hello, welcome to LAB 112!

Methods can also accept input (called parameters) and return output: 

// Method with parameters and return value
static int Add(int a, int b)
{
    return a + b;
}

// Calling and using the result
int result = Add(5, 3);
Console.WriteLine(result); // Output: 8
Classes & Objects
The Blueprint and the Thing Covered

A class is a blueprint or template. An object is something you create from that blueprint. C# is an Object-Oriented Programming (OOP) language — meaning almost everything is built around classes and objects.

Real life example

A class is like an architect's house plan. The plan itself is not a house — it's just a blueprint. When you actually build a house using that plan, that built house is the object. You can build many houses (objects) from one plan (class).

// Defining a class
class Student
{
    public string Name;
    public int Age;

    public void Introduce()
    {
        Console.WriteLine($"Hi, I am {Name} and I am {Age} years old.");
    }
}

// Creating an object from the class
Student student1 = new Student();
student1.Name = "Alice";
student1.Age = 20;
student1.Introduce();

// Output:
Hi, I am Alice and I am 20 years old.

The variables inside a class (like Name and Age) are called fields or properties. The methods inside a class describe what objects of that class can do.

Access Modifiers
Controlling Who Can See What Covered

Access modifiers control which parts of your program can access a class, method, or variable. They are used to protect your data and keep your code organised.

Real life example

Think of a school building. Some rooms are public — anyone can walk in (like the canteen). Some rooms are private — only the teacher can enter (like the staff room). Some are protected — only teachers and their assistants can enter.

class BankAccount
{
    public string OwnerName;    // anyone can access this
    private double balance;     // only this class can access this
    protected int accountId;   // this class and child classes
}

The three most common access modifiers:

public — accessible from anywhere in the program
private — only accessible within the same class
protected — accessible within the class and any class that inherits from it

Without access modifiers

If you had no access modifiers, any part of your program could read or change your bank balance directly — that would be dangerous! Access modifiers keep sensitive data safe.

Inheritance
Passing Down Properties Covered

Inheritance allows one class to take (inherit) the properties and methods of another class. The class being inherited from is called the parent class or base class. The class that inherits is called the child class or derived class.

Real life example

A Dog and a Cat are both Animals. Instead of writing "has a name", "can breathe", "can eat" separately for both, you create one Animal class with those properties — and both Dog and Cat inherit from Animal automatically. They get everything Animal has, plus their own unique things.

// Parent class
class Animal
{
    public string Name;

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

// Child class — inherits from Animal using :
class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} says: Woof!");
    }
}

// Dog has both Eat() from Animal AND Bark() its own
Dog myDog = new Dog();
myDog.Name = "Rex";
myDog.Eat();   // inherited from Animal
myDog.Bark();  // Dog's own method

// Output:
Rex is eating.
Rex says: Woof!
Why inheritance matters

Without inheritance, you would have to rewrite the same code in every class. With inheritance, you write it once in the parent and all children get it automatically. This keeps your code clean, organised and easy to update.

Constructors
Setting Up an Object at Birth Covered

A constructor is a special method that runs automatically when you create an object. Its job is to set up the object with initial values right from the start.

Real life example

When a student joins UICT, they automatically get assigned a student ID, a name on the register, and a course. This "setup when you join" is exactly what a constructor does — it runs the moment the object is created.

class Student
{
    public string Name;
    public int Age;

    // Constructor — same name as the class, no return type
    public Student(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

// Object is created AND set up in one line
Student s1 = new Student("Alice", 20);
Console.WriteLine(s1.Name); // Alice

Without a constructor you would have to manually set every property after creating the object. With a constructor it all happens in one step automatically.

Encapsulation
Protecting Your Data Covered

Encapsulation means hiding the internal details of a class and only exposing what is necessary. You make fields private so they can't be directly changed from outside the class, then provide controlled access through methods.

Real life example

A vending machine is perfectly encapsulated. You don't reach inside and grab a drink directly — you press a button (the public method) and the machine handles everything internally. The internal mechanism is hidden from you. You only interact with what's exposed.

Without encapsulation
class BankAccount
{
  public double balance;
}

// Anyone can change it directly!
account.balance = -99999;
With encapsulation
class BankAccount
{
  private double balance;

  public void Deposit(double amt)
  {
    if (amt > 0)
      balance += amt;
  }
}

With encapsulation, you control exactly how data is changed. Nobody can set the balance to a negative number directly — they have to go through your Deposit method which checks that the amount is valid first.

Getters & Setters
Controlled Access to Private Data Read Ahead

Getters and Setters are special methods that allow you to read (get) or change (set) a private variable in a controlled way. They work directly with encapsulation.

Real life example

A temperature display in a room — you can see the temperature (get) and you can adjust the thermostat (set), but you can't directly reach into the wall and mess with the wiring. The thermostat is the setter — it lets you change the temperature but only within safe limits.

class Student
{
    private int age;

    // Getter — reads the value
    public int GetAge()
    {
        return age;
    }

    // Setter — changes the value with a check
    public void SetAge(int value)
    {
        if (value > 0)
            age = value;
    }
}

Student s = new Student();
s.SetAge(20);
Console.WriteLine(s.GetAge()); // 20

C# also has a cleaner built-in way to write getters and setters using properties:

class Student
{
    private int age;

    public int Age
    {
        get { return age; }
        set { if (value > 0) age = value; }
    }
}

Student s = new Student();
s.Age = 20;             // calls the setter
Console.WriteLine(s.Age); // calls the getter → 20
The void Keyword
Methods That Return Nothing Covered

void is used before a method name to indicate that the method does not return any value. It just performs an action and that's it.

Real life example

A printer prints a document — it just does the job. It doesn't hand you back anything in return. Compare that to a calculator which takes numbers and gives you a result back. The printer is void. The calculator returns a value.

// void — does something, returns nothing
static void PrintMessage()
{
    Console.WriteLine("This method returns nothing!");
}

// int — does something AND returns a number
static int AddNumbers(int a, int b)
{
    return a + b; // must return an int
}

Whenever you see void before a method, you know that calling it won't give you anything back — it just runs its instructions.

Polymorphism
One Name, Many Forms Covered

Polymorphism means "many forms." It allows the same method name to behave differently depending on which class is using it. This is one of the most powerful features of OOP.

Real life example

The word "speak" means something different depending on who does it. A dog speaks by barking. A cat speaks by meowing. A person speaks by talking. Same action — Speak() — completely different behaviour depending on the animal. That's polymorphism.

class Animal
{
    public virtual void Speak()
    {
        Console.WriteLine("Some animal sound...");
    }
}

class Dog : Animal
{
    public override void Speak()
    {
        Console.WriteLine("Woof!");
    }
}

class Cat : Animal
{
    public override void Speak()
    {
        Console.WriteLine("Meow!");
    }
}

Animal a = new Dog();
a.Speak(); // Output: Woof!

Animal b = new Cat();
b.Speak(); // Output: Meow!

The keyword virtual in the parent says "this method can be overridden." The keyword override in the child says "I am replacing this method with my own version."

Method Overloading vs Overriding
Similar Names, Different Behaviours Read Ahead

In C#, methods can share names but behave differently. We use Overloading and Overriding to achieve this.

Method Overloading

Overloading happens when multiple methods in the same class share the same name but have different parameters (different type or number of arguments). It's a compile-time polymorphism.

class Calculator
{
    // Same method name 'Add', but different parameters
    public int Add(int a, int b) {
        return a + b;
    }
    
    public double Add(double a, double b) {
        return a + b;
    }
}
Method Overriding

Overriding happens when a child class provides its own implementation for a method that is already defined in its parent class. It requires the virtual keyword in the parent and override keyword in the child. It's a run-time polymorphism.

You can see an example of Overriding in the Polymorphism section above.

Overloading

Same class, same method name, different parameters.

Overriding

Different classes (parent/child), same method name, same parameters.

The static Keyword
Belonging to the Class, Not the Object Read Ahead

When a method or variable is marked as static, it belongs to the class itself, rather than any specific object created from that class.

Real life example

If you have a Student class, the Name and Age variables belong to each individual student object. But a variable like TotalStudentsEnrolled should be shared across everyone. It belongs to the concept of the Student class, not one person. That's a static variable.

class MathUtils
{
    // Static method
    public static int Square(int number)
    {
        return number * number;
    }
}

// You don't need to create an object to use it!
// MathUtils utils = new MathUtils(); -- Not needed!
int result = MathUtils.Square(5);
Console.WriteLine(result); // 25

A static method cannot access non-static variables directly because it doesn't know which object's variables to look at.

Abstraction
Hiding Complexity, Showing Simplicity Read Ahead

Abstraction means hiding the complex internal workings and only showing what the user needs to see. You define what something should do, not how it does it.

Real life example

When you drive a car, you use the steering wheel, accelerator and brakes. You don't need to understand how the engine, gearbox, or fuel injection work internally — all of that is hidden from you. You only see and use what you need. That's abstraction.

In C#, abstraction is achieved using abstract classes. An abstract class cannot be used to create objects directly — it just defines a template that other classes must follow.

// Abstract class — cannot create objects from this directly
abstract class Shape
{
    // Abstract method — no body, just the rule
    public abstract double GetArea();
}

// Circle must provide its own GetArea
class Circle : Shape
{
    public double Radius;

    public override double GetArea()
    {
        return 3.14 * Radius * Radius;
    }
}

Circle c = new Circle();
c.Radius = 5;
Console.WriteLine(c.GetArea()); // 78.5
Abstraction vs Encapsulation — the difference

These two are often confused. Here is the clearest way to understand them:

Encapsulation = hiding data — protecting your variables so they can't be directly changed from outside. It's about keeping data safe.

Abstraction = hiding complexity — hiding how something works internally so the user only sees a simple interface. It's about keeping things simple to use.

Encapsulation protects. Abstraction simplifies.

Interfaces
A Contract a Class Must Follow Read Ahead

An interface is like a contract. It says "any class that uses me must have these methods." The interface only defines what methods exist — never how they work. The class that uses the interface fills in the how.

Real life example

A job description is an interface. It says "this employee must be able to: write reports, attend meetings, manage a team." It doesn't say how they do it — just that they must. When someone accepts the job (a class implements the interface), they commit to being able to do all those things.

// Interface — defines the contract
interface IAnimal
{
    void Speak();   // no body — just the rule
    void Move();
}

// Dog agrees to the contract using :
class Dog : IAnimal
{
    public void Speak()
    {
        Console.WriteLine("Woof!");
    }

    public void Move()
    {
        Console.WriteLine("Dog runs.");
    }
}

The difference between an interface and an abstract class:

An abstract class can have some methods with code already written and some without. A class can only inherit from one abstract class.

An interface has no method bodies at all — just rules. A class can implement multiple interfaces.

COURSE 02
Java Programming
Introduction to Java
JVM, JRE, and JDK Covered

Java's architecture is based on three main components: the JDK, JRE, and JVM. Together, they allow Java code to run on any device.

Real life example

Imagine a kitchen. The JDK is the full kitchen with tools (compilers) to create recipes. The JRE is the kitchen appliances (libraries) needed to cook. The JVM is the chef who reads the recipe and cooks the dish.

Here is what each stands for:

JDK (Java Development Kit) — Tools you need to write and compile Java code.
JRE (Java Runtime Environment) — The environment required to run Java programs.
JVM (Java Virtual Machine) — The engine that executes the code line-by-line.

Write Once, Run Anywhere

Because the JVM exists for Windows, Mac, and Linux, your Java code is compiled into bytecode. Any JVM can understand this bytecode, which makes Java platform-independent.

Java Fundamentals
Data Types and Variables Covered

Like C#, Java is a strongly typed language. You must declare a variable's data type before assigning it a value.

// Declaring variables in Java
int age = 20;
double price = 19.99;
String name = "Bob"; // String has a capital 'S' in Java!
boolean isOnline = true;

Common data types in Java:

int — whole numbers (e.g. 50)
double — decimal numbers (e.g. 3.14)
String — text (e.g. "Hello"). Note: uppercase S.
boolean — true or false
char — single character (e.g. 'A')

Printing to the Console

Instead of Console.WriteLine() like in C#, Java uses System.out.println() to display output.

System.out.println("Hello from Java!");
// Output:
Hello from Java!
User Input
Reading Data with Scanner Covered

In Java, we use the Scanner class to read input from the user. It is part of the java.util package, so you must import it at the top of your file.

Real life example

Think of a Scanner at a supermarket checkout. It reads the barcode (input) and brings the information into the cash register system. In Java, Scanner reads what you type on the keyboard and brings it into your variables.

import java.util.Scanner; // 1. Import the Scanner class

public class Main {
    public static void main(String[] args) {
        // 2. Create a Scanner object connected to the keyboard
        Scanner scanner = new Scanner(System.in);
        
        System.out.println("Enter your age:");
        
        // 3. Read user input using the correct method
        int age = scanner.nextInt(); 
        
        System.out.println("You are " + age + " years old.");
    }
}

Common Scanner methods:

nextLine() — reads a full string of text (e.g. "John Doe")
next() — reads a single word (e.g. "John")
nextInt() — reads a whole number
nextDouble() — reads a decimal number

Loops
Repeating Actions Covered

A loop executes a block of code multiple times. Using loops keeps your code clean and avoids redundancy.

Real life example

Think of a washing machine's Spin Cycle. It doesn't just spin once; it spins continuously (loops) until the timer hits zero. The condition to stop is when the timer ends.

There are four common ways to loop in Java:

1. The 'for' Loop

Use when you know exactly how many times you want the loop to run. The for loop has three distinct parts that control its flow:

  • Start (int i = 1): This creates a counter variable. It only runs once at the very beginning.
  • Stop (i <= 3): This is the condition. The loop keeps running as long as this is true. It checks this before every loop.
  • Step (i++): This updates the counter (adds 1). It runs at the end of every loop.
// 1. Start at 1 | 2. Check if <= 3 | 4. Increase by 1
for (int i = 1; i <= 3; i++) {
    // 3. Run this code
    System.out.println("Count: " + i);
}
2. The 'while' Loop

Use when the loop should run as long as a condition is true.

int health = 3;
while (health > 0) {
    System.out.println("Still alive!");
    health--; // decreases health
}
3. The 'do-while' Loop

This loop will always run at least once before checking the condition.

int attempts = 0;
do {
    System.out.println("Trying connection...");
    attempts++;
} while (attempts < 1);
4. The 'for-each' Loop

Used exclusively to loop through elements in an array.

String[] cars = {"Volvo", "BMW", "Ford"};
for (String i : cars) {
    System.out.println(i);
}
Advanced Loop Techniques Covered
Nested Loops (Loops inside Loops)

You can place a loop inside another loop. The "inner loop" will execute all of its iterations for every single iteration of the "outer loop".

for (int i = 1; i <= 2; i++) { // Outer loop (runs 2 times)
    System.out.println("Outer: " + i);
    
    for (int j = 1; j <= 3; j++) { // Inner loop (runs 3 times per outer)
        System.out.println("  Inner: " + j);
    }
}
// Total inner runs = 2 x 3 = 6 times

How it works:
1. The outer loop starts with i = 1.
2. It prints "Outer: 1".
3. Then, the inner loop starts. It runs completely from j = 1 to j = 3, printing "Inner: 1", "Inner: 2", and "Inner: 3".
4. The inner loop finishes, and the outer loop moves to its second step (i = 2).
5. The inner loop starts completely over again from 1 to 3. Think of it like hours (outer) and minutes (inner) on a clock.

Reversing a Loop

You don't always have to count forwards using i++. You can count backwards using i--. This is extremely useful for displaying arrays in reverse.

// Counting backwards from 5 down to 1
for (int i = 5; i > 0; i--) {
    System.out.println(i);
}
// Output: 5, 4, 3, 2, 1

How it works:
1. Start: We start with i = 5 instead of 0.
2. Stop: The condition is i > 0. As long as the number is greater than zero, the loop keeps running.
3. Step: Instead of i++ (add 1), we use i-- (subtract 1). It prints 5, subtracts 1 to become 4, prints 4, and so on until it hits 0 and stops.

Interactive Example: Storing Input in an Array

Here is a classic program: We use a Scanner and a loop to ask the user for 3 names, store them in an array, and then use a second loop to print them out.

Scanner scanner = new Scanner(System.in);
String[] students = new String[3];

// 1. Loop to get input and store it
for (int i = 0; i < students.length; i++) {
    System.out.print("Enter student " + (i+1) + " name: ");
    students[i] = scanner.nextLine();
}

System.out.println("\n--- Student List ---");

// 2. Loop to display the stored data
for (String name : students) {
    System.out.println("Student: " + name);
}

How it works:
Loop 1 (Input): We use a for loop to ask the user for a name 3 times. We use students.length to ensure we fill the array perfectly. students[i] stores each name into slot 0, then slot 1, then slot 2.
Loop 2 (Output): We use an enhanced for-each loop. It automatically grabs each string out of the students array one by one, temporarily calls it name, and prints it out.

Reading Complex Loops

Q: How do I trace and understand a complex loop?
A: Never try to guess what a loop does by just glancing at it! Read it like a story. Identify the starting value (int i=0), the stopping condition (i < 5), and how it steps (i++). If the loop contains complex math or nested loops, draw a small "trace table" on a piece of paper and track the variables on every single pass until the loop breaks.

Arrays
Storing Multiple Values Covered

An array is a variable that can hold multiple values of the same type. In Java, arrays have a fixed size — once you create it, you cannot change how many items it holds.

Real life example

An array is like a pill box with fixed compartments for Monday, Tuesday, Wednesday, etc. There are exactly 7 slots. You can put a pill in each slot, but you cannot suddenly add an 8th slot.

// 1. Declaring an array with a specific size
String[] days = new String[3];
days[0] = "Monday";
days[1] = "Tuesday";
days[2] = "Wednesday";

// 2. Shorter way (initializer list)
int[] scores = { 85, 90, 78 };

// Accessing items (index starts at 0)
System.out.println(scores[0]); // Prints 85
System.out.println(scores.length); // Prints 3
Classes & Objects
The Blueprint and the Thing Covered

Everything in Java is associated with classes and objects. A class is a blueprint, and an object is an instance of it.

Real life example

A class is like an architect's house plan. The plan itself is not a house. When you actually build a house using that plan, that built house is the object. You can build many houses (objects) from one plan (class).

class Car {
    String brand;
    
    public void drive() {
        System.out.println(brand + " is driving.");
    }
}

Car myCar = new Car();
myCar.brand = "Toyota";
myCar.drive(); // Toyota is driving.
Constructors
Setting Up an Object Covered

A constructor in Java is a special method that is used to initialize objects. The constructor is called when an object of a class is created.

Real life example

When you buy a new phone, it comes with a default set of apps, a set battery level, and factory settings. The process of setting it up the very first time you turn it on is like a constructor — it gives the object its initial state.

class Car {
    String modelName;

    // Constructor - Must match the class name, no return type
    public Car(String name) {
        modelName = name;
    }
}

Car myCar = new Car("Mustang"); // Constructor is called here
System.out.println(myCar.modelName); // Prints Mustang

Note that constructors do not have a return type, not even void, and they must have the exact same name as the class.

Inheritance
Passing Down Traits Covered

Inheritance allows a new class to take the properties and methods of an existing class using the extends keyword. It promotes code reusability.

Real life example

A Smartphone inherits from a general Phone. The general phone has a "call" feature. The smartphone automatically gets that "call" feature without rewriting it, and adds its own "browse internet" feature.

// Parent class
class Phone {
    public void call() {
        System.out.println("Making a phone call...");
    }
}

// Child class uses 'extends' to inherit
class SmartPhone extends Phone {
    public void browse() {
        System.out.println("Browsing the internet...");
    }
}

SmartPhone myPhone = new SmartPhone();
myPhone.call();   // Inherited from Phone
myPhone.browse(); // SmartPhone's own method
Encapsulation
Protecting Data Covered

Encapsulation hides the internal state of an object and requires all interaction to be performed through an object's methods. We do this by making variables private and providing public getter and setter methods.

Real life example

A bank ATM. You cannot reach inside the machine to change your balance directly (the money is private). You must use the keypad options to deposit or withdraw (the public setters/getters) which safely update your balance.

class Account {
    // 1. Private data
    private double balance;

    // 2. Public Setter (Controls how data changes)
    public void setBalance(double amount) {
        if (amount > 0) {
            this.balance = amount;
        }
    }

    // 3. Public Getter (Allows reading data)
    public double getBalance() {
        return this.balance;
    }
}

Account acc = new Account();
acc.setBalance(1000);
System.out.println(acc.getBalance()); // 1000
Polymorphism
Many Forms Covered

Polymorphism means "many forms". It occurs when we have many classes that are related to each other by inheritance. It allows us to perform a single action in different ways by overriding methods using @Override.

Real life example

A person taking on many roles. At the mall, a person is a Customer. On a bus, the same person is a Passenger. At school, they are a Student. Same person, different behaviours based on the situation.

class Animal {
    public void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Pig extends Animal {
    @Override // This tells Java we are replacing the parent method
    public void sound() {
        System.out.println("The pig says: wee wee");
    }
}

class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("The dog says: bow wow");
    }
}

Animal myAnimal = new Pig();
myAnimal.sound(); // Outputs: The pig says: wee wee
Method Overloading vs Overriding
Similar Names, Different Behaviours Covered

Java supports both Overloading and Overriding. While C# uses the virtual and override keywords for overriding, Java methods are virtual by default.

Method Overloading

Overloading happens when multiple methods in the same class share the same name but have different parameters. It's compile-time polymorphism.

class Calculator {
    // Overloaded methods
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }
}
Method Overriding

Overriding happens when a child class provides a specific implementation of a method that is already provided by its parent class. We use the @Override annotation to ensure we are correctly replacing the parent's method.

In Java, you don't need a virtual keyword in the parent class to override a method. You just declare it, and the child class can override it using @Override.

The static Keyword
Belonging to the Class Covered

The static keyword in Java means that the particular member belongs to the type itself, rather than to an instance of that type.

Real life example

If every student in a school has their own ID, that's non-static. But if there's a total count of all students ever enrolled in the school, that belongs to the whole school, not one student. That would be a static variable.

class MathUtils {
    // Static method
    public static int square(int number) {
        return number * number;
    }
}

// Calling a static method without creating an object
int result = MathUtils.square(5);
System.out.println(result); // 25

A common example of a static method in Java is the public static void main(String[] args) method. It is static because the JVM needs to call it before any objects are created.

Abstraction
Hiding Complexity Covered

Abstraction is the process of hiding certain details and showing only essential information to the user. We achieve this with abstract classes.

Real life example

Sending an email. You just type the message, click "Send", and it delivers. You don't see the complex server routing, protocol handshakes, or data packaging happening in the background. That complexity is abstracted away.

// Abstract class: restricted class that cannot be used to create objects
abstract class Animal {
    // Abstract method: does not have a body
    public abstract void animalSound();
    
    // Regular method
    public void sleep() {
        System.out.println("Zzz");
    }
}

// Subclass (inherit from Animal)
class Pig extends Animal {
    public void animalSound() {
        // The body of animalSound() is provided here
        System.out.println("The pig says: wee wee");
    }
}
Interfaces
Complete Abstraction Covered

An interface is a completely "abstract class" that is used to group related methods with empty bodies. A class uses the implements keyword to inherit the interface.

Real life example

A USB port is an interface. It defines a standard connection shape. Any device (mouse, keyboard, flash drive) that wants to plug into your laptop must implement that exact USB shape. The laptop doesn't care what the device does, only that it perfectly matches the interface.

interface Animal {
    public void animalSound(); // interface method (does not have a body)
    public void run(); // interface method (does not have a body)
}

// Pig "implements" the Animal interface
class Pig implements Animal {
    public void animalSound() {
        System.out.println("The pig says: wee wee");
    }
    public void run() {
        System.out.println("The pig is running");
    }
}
Packages
Organizing Your Code Covered

A package in Java is used to group related classes. Think of it as a folder in a file directory. We use packages to avoid name conflicts, and to write a better maintainable code.

Real life example

Imagine your computer's hard drive. Instead of putting all your photos, documents, and videos into one giant folder, you group them into separate folders like /Pictures, /Documents/Work, and /Videos/Vacation. Packages do the exact same thing for Java classes.

Packages are divided into two categories:

  • Built-in Packages (packages from the Java API, like java.util, java.io, java.sql)
  • User-defined Packages (create your own packages to organize your project)
Using Built-in Packages

To use a class or a package from the library, you need to use the import keyword.

// Import a single class
import java.util.Scanner;

// Import the whole package (all classes in java.util)
import java.util.*;

class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.println("Ready!");
    }
}
Creating Your Own Packages

To create a package, you use the package keyword at the very top of your Java file.

// At the top of your file 'MyClass.java'
package mypack;

class MyClass {
    public static void main(String[] args) {
        System.out.println("This is my package!");
    }
}

When saving the file, you should save it in a directory that matches the package name (e.g., inside a folder named mypack). To compile and run it, you would navigate to the root folder in the terminal and compile with javac mypack/MyClass.java, and then run it with java mypack.MyClass.

Exception Handling
Try-Catch Blocks Covered

Exceptions are errors that occur while the program is running. We handle them using try-catch blocks to prevent the program from crashing.

try {
    int result = 10 / 0; // This causes an error
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero!");
} finally {
    System.out.println("This block always runs.");
}
I/O Streams
Reading and Writing Files Covered

Java uses Streams to read from and write to files. A common approach for text files is using Scanner for reading and FileWriter for writing.

import java.io.FileWriter;
import java.io.IOException;

try {
    FileWriter writer = new FileWriter("notes.txt");
    writer.write("Java I/O is powerful.");
    writer.close();
} catch (IOException e) {
    System.out.println("An error occurred.");
}
